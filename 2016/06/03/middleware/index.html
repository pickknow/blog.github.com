中间件meddleware

接下来我们看看中间件是如何来优雅的处理这些问题的，这里我们以laravel来进行说明，lavavel对中间件的说明如下

HTTP 中间件提供一个方便的机制来过滤进入应用程序的 HTTP 请求，例如，Laravel 默认包含了一个中间件来检验用户身份验证，如果用户没有经过身份验证，中间件会将用户导向登录页面，然而，如果用户通过身份验证，中间件将会允许这个请求进一步继续前进。
当然，除了身份验证之外，中间件也可以被用来执行各式各样的任务，CORS 中间件负责替所有即将离开程序的响应加入适当的响应头，一个日志中间件可以记录所有传入应用程序的请求。 Laravel 框架已经内置一些中间件，包括维护、身份验证、CSRF 保护，等等

自然第一个好处是实现了逻辑分离与解耦，oop编程的第一要务就是要实现业务间的结构，看看面向对象编程的第一要素

单一职责原则（SRP）

一个类应该有且只有一个变化的原因。
There should never be more than one reason for a class to change.

这么做的目的是什么呢？目的除了让我们的代码更加清晰，利于测试之外也要实现面向对象编程的另一个要素

开放封闭原则（OCP）

既开放又封闭，对扩展是开放的，对更改是封闭的！
扩展即扩展现行的模块，当我们软件的实际应用发生改变时，出现新的需求，就需要我们对模块进行扩展，使其能够满足新的需求！

对比以上两种方式，如果用继承来解决的话，首先就不符合SRP原则。另外在需要进行业务调整，逻辑修改的时候就更加不方便了，需要修改类的业务执行逻辑，而一点项目复杂起来之后，能保证每一次的修改都不会有遗漏的子类吗，同时每一次的修改都要进行反复测试，已保证程序不会出错。

你看，我们只是调整一下用户访问的限制问题就可以看出两种方法哪个更优雅，利用中间件我们只需要配置路由参数就可以做到灵活运用，完全不需要去修改已经写好的类，非常符合开闭原则。

当然中间件的威力远远不止于此，要知道一个中间件不但可以在请求进入的时候执行逻辑，同时也可以在请求之后进行一些逻辑处理，来看看lavarel的中间件定义。

关于更多的一些中间件的知识 大家可以自己google搜索一下，其实中间件的思路 我们以前也都接触过 比如 AOP 面向横切面编程，hook机制，都是为了解决面向对象编程中日益庞大的继承，以及业务越来越负责的场景。
现在一些新的框架比如koa 已经完全使用中间件来处理
